REX ETP Tracker API

Programmatic access to ETP screener data, SEC filing records, and total return comparisons from the REX Financial Intelligence Hub. All responses are JSON.

Authentication

All API requests require an X-API-Key header. Contact the REX team to request an API key. 

X-API-Key: your-api-key-here
Keep your API key private. Do not expose it in client-side code or public repositories.

Base URL

{{ base_url }}/api/v1

Quick Start

Here are examples of how to call the API in different languages.

# Get all REX products
curl -H "X-API-Key: YOUR_KEY" \
  "{{ base_url }}/api/v1/etp/screener?scope=rex"

# Get recent 485BPOS filings from the last 7 days
curl -H "X-API-Key: YOUR_KEY" \
  "{{ base_url }}/api/v1/filings/recent?days=7&form=485BPOS"
import requests

API_KEY = "YOUR_KEY"
BASE    = "{{ base_url }}/api/v1"
headers = {"X-API-Key": API_KEY}

# Get all REX products
resp = requests.get(f"{BASE}/etp/screener?scope=rex", headers=headers)
data = resp.json()

for fund in data["funds"]:
    print(fund["ticker"], fund["aum"])
const API_KEY = "YOUR_KEY";
const BASE    = "{{ base_url }}/api/v1";

const res = await fetch(`${BASE}/etp/screener?scope=rex`, {
  headers: { "X-API-Key": API_KEY }
});
const data = await res.json();

console.log(`${data.count} funds, total records`);
data.funds.forEach(f => console.log(f.ticker, f.aum));
GET /api/v1/etp/screener

Full ETP screener data with 50+ fields for every active ETF and ETN. Covers AUM, returns, flows, expense ratios, leverage details, and category mappings.

Parameters

NameTypeDefaultDescription
scope string all all, rex (REX products only), or competitors (non-REX)
ticker string -- Comma-separated tickers, e.g. NVDX US,TSLL US
category string -- Filter by ETP category: LI, CC, Crypto, Defined, Thematic
limit int 0 Max results to return. 0 = all matching funds.
Example Response

  "funds": [
    {
      "ticker": "NVDX US",
      "fund_name": "T-REX 2X Long NVIDIA Daily Target ETF",
      "issuer_display": "REX Financial",
      "aum": 1250.43,
      "market_status": "ACTV",
      "etp_category": "LI",
      "total_return_1day": 0.0342,
      "total_return_ytd": 0.1856,
      "expense_ratio": 1.05,
      "fund_flow_1month": 45.2,
      "inception_date": "2023-10-18",
      "is_singlestock": 1,
      "uses_leverage": 1,
      "leverage_amount": 2.0,
      // ... 40+ additional fields
    }
  ],
  "count": 847,
  "scope": "all"
}

REX product summary with aggregate KPIs: total AUM, fund count, flows, and the best 1-day performer.

GET /api/v1/returns

Total return comparison data for up to 10 symbols. Returns price series, drawdowns, growth of $10K, annual returns, and summary statistics.

Parameters

NameTypeDefaultDescription
symbols required string -- Comma-separated tickers, e.g. NVII,NVDY,JEPI. Max 10 symbols.
start string -- Start date in YYYY-MM-DD format. Defaults to earliest available data.
end string -- End date in YYYY-MM-DD format. Defaults to latest available data.
Example Response
{
  "symbols": ["NVII", "NVDY"],
  "start": "2024-01-02",
  "end": "2026-04-04",
  "prices": {
    "dates": ["2024-01-02", "2024-01-03", "..."],
    "NVII": [100.0, 101.2, "..."],
    "NVDY": [100.0, 99.8, "..."]
  },
  "growth_10k": {
    "NVII": [10000, 10120, "..."],
    "NVDY": [10000, 9980, "..."]
  },
  "drawdowns": { /* same shape as prices */ },
  "annual_returns": {
    "NVII": {"2024": 0.352, "2025": 0.128},
    "NVDY": {"2024": 0.221, "2025": 0.094}
  },
  "stats": {
    "NVII": {"total_return": 0.486, "max_drawdown": -0.312, "..."},
    "NVDY": {"total_return": 0.321, "max_drawdown": -0.187, "..."}
  }
}

Rate Limits

There is no formal rate limit enforced on the API. That said, please be reasonable with request volume. If you need to poll for updates, once per minute is more than sufficient -- the underlying data refreshes daily.

ETP screener data and filings update once daily. Total returns data is sourced live per request.

Errors

The API uses standard HTTP status codes. Error responses include a JSON body with a detail field.

CodeMeaning
400Bad request -- missing or invalid parameters
401Invalid or missing API key
404Resource not found (e.g. unknown symbol in returns)
502Upstream data fetch failed (total returns)
503API not configured (server-side key missing)
Error Response Format
{
  "detail": "Invalid API key"
}
